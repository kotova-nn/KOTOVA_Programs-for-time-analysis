# Сервер генерации персонализированных отчётов

## Описание проекта

Сервер предназначен для автоматической генерации персонализированных отчётов на основе данных об активности и успеваемости учеников. Система поддерживает два типа отчётов: для начальной школы (с игровой механикой «Путь динозаврика Дино») и для старшей школы (с аналитикой активности по дням недели).

## Структура проекта

Проект состоит из следующих файлов и папок: `app.py` — главный сервер (Flask API), `certificate_generator.py` — генератор отчётов, `database.py` — модель базы данных (SQLAlchemy), `students.csv` — датасет с данными учеников, папка `templates/` — PNG-шаблоны для отчётов, папка `share_cache/` — кеш сгенерированных отчётов, `requirements.txt` — зависимости Python, `runtime.txt` — версия Python для Render.com, и `README.md` — документация.

## Описание скриптов

### 1. `app.py` — Главный сервер (Flask API)

Этот файл является веб-сервером, который обрабатывает HTTP-запросы от платформы и управляет всем процессом создания отчётов. Основные функции: приём и обработка API-запросов, загрузка и валидация CSV-файла с данными учеников, генерация уникальных UUID-токенов для каждой ссылки, сохранение токенов и данных учеников в базе данных, доступ к HTML-страницам с отчётами, управление датасетом (добавление, обновление, удаление учеников). Ключевые эндпоинты: POST /api/share/generate для создания ссылки на сертификат, GET /api/student/<id> для получения данных ученика за период, GET /api/student-available-periods/<id> для получения списка доступных периодов, GET /api/students для получения списка всех учеников, POST /api/students/add для добавления нового ученика, PUT /api/students/update/<id> для обновления данных ученика, POST /api/students/upload-csv для загрузки датасета, GET /share/<token> для просмотра сертификата, GET /api/download-pdf/<token> для скачивания PDF-версии отчёта.

### 2. `certificate_generator.py` — Генератор PNG/PDF отчётов

Этот файл создаёт визуальные отчёты (PNG и PDF) на основе данных ученика и выбранного шаблона. Основные функции: загрузка фоновых шаблонов из папки `templates/`, отрисовка текста на изображении (имя, класс, баллы, задачи, минуты), автоматическое масштабирование шрифта для гарантированного размещения текста, расчёт уровня (локации) для начальной школы на основе XP, определение титула и слогана для старшей школы (по дню недели или по типу активности при нескольких днях), сохранение результата в PNG и PDF. Для начальной школы (1-4 класс) создаётся игровой отчёт «Путь динозаврика Дино». Для старшей школы (5-11 класс) создаётся аналитический дашборд с персонализированными титулами: для одного активного дня — титул по дню недели, для нескольких дней — титул из колонки `activity_type` (Король будней или Гибкий поток).

### 3. `database.py` — Модель базы данных (SQLAlchemy)

Этот файл определяет структуру базы данных для хранения токенов и метаданных отчётов. Основные функции: создание таблицы `share_tokens` в SQLite (или PostgreSQL), хранение уникальных токенов для каждой ссылки, хранение зафиксированных данных ученика в JSON-формате (`student_data_json`), хранение метаданных (тип периода, номер периода, год, возрастная группа, класс), управление статусом ссылки (активна или отозвана), автоматическое создание базы данных при первом запуске. Таблица `share_tokens` содержит поля: `id` (первичный ключ), `token` (уникальный UUID токен), `student_name` (имя и фамилия ученика), `student_data_json` (JSON с фиксированными данными ученика), `period_type` (тип периода: year или trimester), `period_number` (номер периода), `period_year` (учебный год), `age_group` (возрастная группа: primary или senior), `grade` (класс), `is_active` (активна ли ссылка), `created_at` (дата создания), `views_count` (счётчик просмотров — увеличивается при каждом открытии ссылки). Модель также содержит методы `is_valid()` для проверки активности ссылки и `increment_views()` для увеличения счётчика просмотров.

### 4. `students.csv` — Датасет с данными учеников

Этот файл хранит исходные данные об успеваемости и активности учеников. Колонки датасета: `user_id` (уникальный ID ученика), `last_name` (фамилия), `first_name` (имя), `wk_solved_task_count` (количество решённых задач за период), `wk_points` (набранные баллы за период), `wk_max_task_count` (максимум задач в периоде), `wk_max_points` (максимум баллов в периоде), `grade` (класс от 1 до 11), `period_type` (тип периода: year или trimester), `period_number` (номер периода), `period_year` (учебный год), `favorite_day` (самый продуктивный день недели; если дней несколько — значения через запятую), `activity_type` (тип активности: Король будней или Гибкий поток; используется при нескольких активных днях), `total_events` (количество уникальных дней активности), `total_minutes` (общее время на платформе в минутах).

## Взаимодействие скриптов

Платформа отправляет POST-запрос на эндпоинт `/api/share/generate` в `app.py`. Сервер читает данные ученика из файла `students.csv`, сохраняет их в базу данных через `database.py`, вызывает генератор отчётов `certificate_generator.py`, который создаёт файлы в папке `share_cache/`. Затем сервер возвращает платформе уникальную ссылку. Когда ученик открывает эту ссылку, `app.py` берёт зафиксированные данные из базы данных `database.py`, увеличивает счётчик просмотров, проверяет наличие отчёта в кэше (чтобы не перегенерировать при повторных открытиях) и показывает PNG-изображение с возможностью скачать PDF и осуществить шеринг.

## Запуск проекта

Для локального запуска выполните команду `python app.py` в терминале. Сервер будет доступен по адресу `http://localhost:5001`. Тестовая страница для разработчиков доступна по адресу `http://localhost:5001/test`.

## Зависимости

Версии зависимостей указаны в файле `requirements.txt`: Flask версии 2.3.2, SQLAlchemy версии 2.0.23, Pillow версии 10.0.0, Gunicorn версии 21.2.0.
